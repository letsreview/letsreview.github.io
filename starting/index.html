Getting Started

September 26, 2022

If you are unable to join an existing group or want to get into code review on your own, then the large amount of possibilities might be overwhelming. Especially if you are very new to the subject, you might not even know the correct terms to use in a search engine to figure out how to get started.

In such a case, this report might help you with your first steps.

What is Your Motivation?#

This might sound trivial: “I want to review code!” But why do you want to do that? The Q&A might look like this:

  • I want to review code. Why?
  • I want to improve my software development skill. Why?
  • I want to be able to sustain myself.

This might look reasonable at first, but try to approach these answers from the opposite direction:

  • I want to be able to sustain myself. How?
  • By improving my software development skill. How?
  • By reviewing code.

You can see that suddenly it sounds a bit off. Sustaining myself by improving my skills? It sounds more reasonable that I want to sustain myself by getting paid for working as a software developer and for THIS I have to improve skills.

Also, try to challenge these answers to see if they are really the right choices for you. I could replace “reviewing code” with “writing new code” and see that it sounds reasonable, too.

Try to prioritize conflicting possibilities, e.g. “writing new code is more important to me than reviewing code so I would spend more time writing new code.”

Which Software?#

Computers are very complex devices and the system which you use to read these lines of text consists of hundreds or probably thousands of tools and projects.

If you write software on your own, there is a good chance that you use frameworks or libraries.

My advice is to concentrate on things you use. Either projects which are installed on your system or which you regularly use for development.

Do you use Chrome as a browser? Or maybe Firefox? If you do not want to get right into their code bases, look at their dependencies. Or look at the dependencies both have in common.
